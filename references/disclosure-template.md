# 技术交底书撰写模板与指南

## 文档格式

技术交底书是发明人提供给专利代理人的技术方案说明材料。代理人根据交底书撰写正式的专利申请文件。交底书质量直接影响专利申请的质量和保护范围。

---

## 模板结构

### 头部信息

```
技术方案名称：[简明扼要，体现核心技术特征]
技术领域：[所属技术领域]
技术联系人：[姓名]          联系方式：[电话/邮箱]
```

---

### 第1部分：简要说明本技术的目的以及基本方案

**要求：** 简单地通过几句话或一小段介绍下方案和目的。

**写法指导：**
- 用1-3句话概括发明的核心目的
- 再用1小段描述基本技术方案
- 不需要展开细节，让读者快速了解全貌

**示例（AI方向）：**
```
本技术旨在解决现有多模态数据融合方法中信息损失严重的问题，提出一种基于跨尺度
注意力机制的特征融合方法。该方法通过在不同空间尺度上提取特征并利用注意力权重
进行自适应加权融合，在保持计算效率的同时显著提升了融合特征的表达能力。
```

---

### 第2部分：相关术语解释

**要求：** 对本交底书中不常见或有歧义的术语做下解释。通过百度百科等简单检索可得到的术语可不解释。

**写法指导：**
- 列出本领域的专业术语
- 对每个术语给出简洁的中文定义
- 对外文缩写必须给出中文全称和解释

**示例：**
```
- Transformer：一种基于自注意力机制的深度学习模型架构，通过计算输入序列中各
  元素之间的相关性权重来实现全局特征建模。
- 跨尺度注意力（Cross-Scale Attention）：一种在不同空间分辨率的特征图之间
  计算注意力权重的机制，用于融合多尺度信息。
- 特征金字塔网络（Feature Pyramid Network, FPN）：一种多尺度特征提取结构，
  通过自底向上和自顶向下的路径生成多个尺度的特征图。
```

---

### 第3部分：详细介绍技术背景和现有技术

**要求：**
1. 介绍现在存在什么技术问题或技术缺陷
2. 别人是怎么做的（现有技术方案）
3. 别人的做法有什么缺点

**关键规则：**
- 缺点应当是**技术上的缺点**（如成本高、稳定性差、精度低、计算量大等）
- **不能直接下结论**，需要有简单的推理过程以得出缺点
- 可以附带相关论文、专利等参考文献

**写法指导：**
- 从一般人了解的事物出发，逐步引入专业领域
- 通过中间环节将读者引入发明人所面临的技术问题
- 介绍现有方案时，说明其功能和特点
- 分析缺点时，提供技术层面的推理

**示例结构：**
```
[技术背景概述 — 从宽泛到具体]

随着医学影像技术的发展，...（背景引入）

目前，解决该问题的主要方法有：

方案一：基于传统CNN的方法。例如ResNet（He et al., 2016）通过残差连接实现
了深层网络的训练。但该方案采用固定大小的卷积核，感受野范围有限。当应用于
医学图像时，由于病变区域大小不一，固定感受野难以同时捕获局部细节和全局
上下文信息，导致在小目标病变检测上准确率下降约15%。

方案二：基于Vision Transformer的方法。ViT（Dosovitskiy et al., 2021）
通过自注意力机制实现了全局特征建模。然而，自注意力的计算复杂度为O(n²)，
当输入图像分辨率为512×512时，所需的计算量和显存占用显著增加，使得该方案
难以在临床常用的中等算力设备上实时运行。

因此，如何在保持计算效率的同时有效融合多尺度特征，成为亟需解决的技术问题。
```

---

### 第4部分：本技术方案的详细阐述 ★最重要★

**要求：**
- 此部分最重要，**越详细越好**，建议至少提供3页
- 介绍清楚本发明的具体技术方案
- 详尽程度：让理工科初级技术人员能看懂
- 不能只有原理，也不能只作功能介绍
- 须结合附图

**附图要求：**
- 方法专利 → 提供**流程图**
- 机械结构 → 整体图、局部图、细节图、多角度图、爆炸图
- 附图以方框图、黑白线条图方式提供
- 附图中各部件有唯一标号
- 每幅附图都有对应的文字描述
- 同一部件全文统一名称和标号

**写法指导（AI/软件方向）：**

```
[整体架构描述]
如图1所示，本技术方案的整体架构包括以下模块：
- 数据预处理模块（101）
- 多尺度特征提取模块（102）
- 跨尺度注意力融合模块（103）
- 分类输出模块（104）

[各模块详细描述]

一、数据预处理模块（101）
输入医学图像后，首先进行以下预处理操作：
（1）将图像统一缩放至224×224像素；
（2）对像素值进行归一化，使其分布在[0,1]区间内；
（3）进行数据增强，包括随机水平翻转（概率0.5）和随机旋转（±15度）。

二、多尺度特征提取模块（102）
如图2所示，该模块包含三个并行的特征提取分支...

[训练过程描述 — AI方向需要]
模型训练过程如下：
- 训练数据集：包含X万张标注图像，涵盖Y个类别
- 损失函数：采用交叉熵损失与对比学习损失的加权组合，权重分别为0.7和0.3

[实验数据/效果 — 如有]
```

### 第5部分：用推理方式推导出本技术方案的优点

**要求：** 写出本方案的优点或效果，需要有简单的推导过程，介绍**为什么**会有这些优点。

**关键规则：** 不是直接列优点，而是通过推理说明优点的来源。

**示例：**
```
本技术方案具有以下优点：

1. 多尺度特征融合能力强：由于采用了三个不同下采样倍率（4x、8x、16x）的并行
特征提取分支，不同分支的感受野范围不同，因此能够同时捕获局部细节特征和全局
上下文信息。相比单一尺度方法，分类准确率提升了5.3%。

2. 计算效率高：由于跨尺度注意力机制仅在不同尺度的特征图之间计算注意力权重，
而非在全部像素位置上计算，因此计算复杂度从O(n²)降低为O(k·n)（其中k为尺度
数量），相比标准Transformer自注意力机制计算量降低约60%。

3. 泛化性好：由于多尺度特征提取不依赖特定数据集的先验知识，而是通过自适应
注意力权重学习不同尺度特征的重要性，因此在不同类型的医学图像上均表现稳定。
```

---

### 第6部分：本技术方案的关键点和欲保护点

**要求：** 简单几句话列出关键点，不需要展开。如果不清楚可不写。

**示例：**
```
本方案的关键技术点包括：
1. 三分支并行多尺度特征提取结构
2. 跨尺度注意力权重计算与自适应融合机制
3. 线性复杂度的注意力计算方式
```

---

### 第7部分：替代方案

**要求：** 是否有其他方式能达到同样的目的？如有请写明，有利于扩大专利保护范围。可不写。

**示例：**
```
针对上述技术方案，以下替代方案同样可以实现类似效果：

替代方案一：将三个并行分支替换为级联结构，即先提取细尺度特征，再逐步下采样
提取粗尺度特征，最后进行融合。该方案计算量略低，但可能损失部分细尺度信息。

替代方案二：将跨尺度注意力机制替换为可变形卷积（Deformable Convolution），
通过学习偏移量实现自适应感受野调整，同样能达到多尺度信息融合的效果。
```

## 撰写注意事项

1. **所有外文必须有中文译文**，尽量避免直接使用外文单词。如必须使用，在第2部分术语解释中给出
2. **全文名称统一**：同一事物只用同一名称，避免一物多名（如"处理器"和"CPU"混用）
3. **技术缺点需推理**：不能直接说"现有方案不好"，需要说明为什么不好
4. **附图标号唯一**：每个部件一个标号，全文一致
5. **第4部分是核心**：详细程度直接决定专利申请质量，宁多勿少
6. **区分方法和结构**：方法类发明用流程图+步骤描述；结构类发明用结构图+组件描述

## 交底书与正式专利的关系

```
交底书各部分  →  对应的正式专利文件部分
─────────────────────────────────────────
第1部分       →  摘要
第2部分       →  说明书正文（术语定义）
第3部分       →  背景技术
第4部分       →  具体实施方式 + 附图说明
第5部分       →  发明内容（有益效果）
第6部分       →  权利要求书（保护范围参考）
第7部分       →  具体实施方式（替代实施例）
```
