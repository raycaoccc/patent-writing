# CNIPA Patent Formatting Guide

## Document Submission Requirements

### Title Page (请求书)
- Invention name (max 25 Chinese characters)
- Applicant name and address
- Inventor name(s)
- Patent agent information (if applicable)
- Priority claim information (if applicable)

### Specification (说明书) Section Format

Each section must use the exact heading:

```
技术领域
背景技术
发明内容
附图说明
具体实施方式
```

### Technical Field (技术领域) Examples

**Good:**
```
本发明涉及人工智能技术领域，具体涉及一种基于Transformer架构的多模态数据融合方法。
```

**Bad (too vague):**
```
本发明涉及计算机技术领域。
```

### Background Technology (背景技术) Writing Rules

1. Cite at least 1-2 prior art references with publication details
2. Objectively describe existing solutions
3. Identify specific technical limitations (not general complaints)
4. Do NOT hint at or describe your solution

**Example template:**
```
[技术背景概述]。目前，现有技术中主要采用[现有方案A]和[现有方案B]的方式来解决[技术问题]。

例如，[参考文献1]（公开号：CNXXXXXXXA，公开日：XXXX年XX月XX日）公开了一种[简述]，但该方案存在[具体不足1]的问题。

又如，[参考文献2]（发表于[期刊/会议]，XXXX年）提出了[简述]，然而该方案仍存在[具体不足2]的缺陷。

因此，如何[待解决的核心问题]，成为本领域亟需解决的技术问题。
```

### Invention Content (发明内容) Structure

Must contain three sub-sections:

**1. Technical Problem (技术问题):**
```
针对现有技术中存在的[具体问题]，本发明提供一种[发明名称]，以解决[核心技术问题]。
```

**2. Technical Solution (技术方案):**
```
为实现上述目的，本发明采用以下技术方案：

[与独立权利要求一致的技术方案描述]
```

**3. Beneficial Effects (有益效果):**
```
与现有技术相比，本发明具有以下有益效果：
1. [效果1，需有数据或理论支撑]；
2. [效果2]；
3. [效果3]。
```

### Drawing Descriptions (附图说明) Format

```
图1为本发明实施例提供的[XXX]的整体架构示意图；
图2为本发明实施例提供的[XXX]的流程图；
图3为本发明实施例提供的[XXX]的结构框图；
图4为本发明实施例中[XXX]的实验对比图。
```

### Detailed Implementation (具体实施方式) Requirements

1. At least 2-3 specific embodiments
2. Reference drawing numbers consistently (如图1所示)
3. Include specific numerical parameters and ranges
4. Describe each step in detail for method claims
5. For AI/ML: include dataset description, model parameters, training configuration, experimental results

**Embodiment template:**
```
下面结合附图和具体实施方式对本发明作进一步详细的说明。

实施例一：
如图1所示，本发明提供的[发明名称]，包括以下步骤：

步骤S1：[具体步骤描述，含参数]；
步骤S2：[具体步骤描述，含参数]；
...

在本实施例中，[具体参数设置]。例如，[参数A]设置为[值]，[参数B]设置为[值]。

实施例二：
与实施例一不同的是，本实施例采用[替代方案]...
```

### Claims (权利要求书) Format

**Independent claim:**
```
1. 一种[发明名称]，其特征在于，包括：
[技术特征A]；
[技术特征B]；以及
[技术特征C]。
```

**Dependent claim:**
```
2. 根据权利要求1所述的[发明名称]，其特征在于，所述[技术特征A]具体为[更具体的描述]。
```

### Abstract (摘要) Format

- Max 300 Chinese characters
- Must include: technical problem, solution, main use/application
- May designate one representative drawing (摘要附图)
- No promotional language

**Template:**
```
本发明公开了一种[发明名称]，属于[技术领域]。所述方法/装置包括：[核心技术方案简述]。与现有技术相比，本发明[主要优势]，可应用于[应用场景]。
```

## Fee Structure (2024)

| Item | Invention Patent | Utility Model |
|------|-----------------|---------------|
| Filing fee | 900 RMB | 500 RMB |
| Substantive exam fee | 2500 RMB | N/A |
| Claims >10, per extra claim | 150 RMB | 150 RMB |
| Annual fee (Year 1-3) | 900 RMB | 600 RMB |

Note: Fee reduction available for individuals and small enterprises (up to 85% reduction).

## Common Rejection Reasons

1. **Lack of novelty** (Article 22.2) - Same features found in single prior art
2. **Lack of inventive step** (Article 22.3) - Obvious combination of known features
3. **Insufficient disclosure** (Article 26.3) - Cannot reproduce from description
4. **Claims not supported** (Article 26.4) - Claims exceed what specification describes
5. **Unclear claims** (Article 26.4) - Ambiguous claim language
